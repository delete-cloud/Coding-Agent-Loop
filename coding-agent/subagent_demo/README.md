# Subagent 概念演示

> 说明：本目录是 **静态概念示例**，用于说明适合交给 subagent 的任务拆分方式。
> 这里的 Markdown 结果是人工整理的示例输出，不是当前仓库内自动跑出来的真实执行记录。

## 示例设计

示意把同一段文本拆成 3 个相互独立的分析任务，分别交给不同的子代理处理。

### 输入文本
```
苹果公司(AAPL)今日股价收于 185.92 美元，上涨 2.3%。
首席执行官 Tim Cook 在发布会上表示："我们对本季度业绩非常满意。"
联系方式：investor@apple.com
微软(MSFT)股价为 378.91 美元，市值约 2.8 万亿美元。
联系方式：ir@microsoft.com
```

### 子代理任务分配

| 代理 | 任务 | 独立性 |
| --- | --- | --- |
| Agent 1 | 文本统计（字符、单词、句子数） | ✅ 独立 |
| Agent 2 | 数字提取（价格、涨幅、市值） | ✅ 独立 |
| Agent 3 | 模式识别（邮箱、股票代码、人名） | ✅ 独立 |

当前仓库里的 `subagent` 工具已经支持：

- 在独立 child pipeline 中执行子任务
- 复用父级 provider 和 session
- 把子任务 trace 追加回父 tape
- 禁止 child 再次暴露 `subagent`，避免递归嵌套

但这个目录 **不代表当前产品已经提供“多 subagent 真并行编排器”**。

---

## Subagent 使用方式（概念层面）

```python
# 在一次工具调用中，把一个清晰独立的子任务交给 subagent
subagent(goal="分析这段文本中的数字信息，并给出结构化总结")
```

如果未来要做“多个子代理并行编排”，还需要额外的调度层来负责：

- 多个 subagent 调用的启动与回收
- 结果聚合
- 失败/超时策略
- 并发预算控制

### 使用场景

1. **独立子问题下钻** - 先把某个子问题交给 child turn 处理
2. **隔离上下文探索** - 避免当前主线被细节搜索污染
3. **结果回流主线** - 子任务产出的 trace 可回收到父 tape

### 适用条件

✅ 子任务目标清晰  
✅ 不依赖与其他子任务实时共享状态  
✅ 子任务完成后可以回到主线继续决策  

### 不适用场景

❌ 需要多个 child 之间实时协商  
❌ 强依赖前一个 child 的中间结果  
❌ 期待当前实现直接提供多 child 并发编排
