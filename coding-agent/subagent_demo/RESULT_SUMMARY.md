# 🧪 Subagent 概念拆分示例汇总

> 说明：这是静态示例文档，用来解释哪些任务适合下放给 subagent。
> 下列“Agent 1/2/3”结果是示例输出，不是当前测试或 CLI 自动生成的真实运行记录。

## 示例设计

这里示意把同一段文本拆成 3 个独立分析视角，展示一种适合 subagent 的任务切分方式。

---

## 任务拆分示意

```
输入文本
  ├─ Agent 1：文本统计
  ├─ Agent 2：数字提取
  └─ Agent 3：模式识别
             ↓
          汇总结果
```

---

## 各子任务示例结果

### Agent 1: 文本统计
- 字符数: 127
- 单词数: 52
- 句子数: 5
- 标点符号: 12

### Agent 2: 数字提取
- 185.92 (苹果股价)
- 2.3% (苹果涨幅)
- 378.91 (微软股价)
- 2.8 (微软市值万亿)

### Agent 3: 模式识别
- 邮箱: investor@apple.com, ir@microsoft.com
- 股票代码: AAPL, MSFT
- 人名: Tim Cook

---

## 📈 汇总洞察

### 标的信息
- 涉及 2 家知名科技公司
- 微软市值（2.8万亿）远超苹果表明的股价
- 两家公司均提供投资者关系渠道
- 高管直接发声，显示重要性

### 设计启发

| 关注点 | 说明 |
|----------|----------|
| 任务独立性 | 每个子任务都应能单独描述、单独完成 |
| 状态隔离 | child turn 不应依赖其他 child 的实时共享状态 |
| 结果汇总 | 主线需要能消费 child 返回的信息或 trace |

---

## 🚀 当前实现对应关系

当前仓库里的 `subagent` 工具更接近下面这种能力：

```python
subagent(goal="""
请在独立 child pipeline 中分析这段文本里的数字信息，
并返回简短结论。
""")
```

如果要把本目录的 3 个子任务真正做成“并行 orchestrator 演示”，还需要额外实现：

- 多次 subagent 调用的调度层
- 聚合器或汇总器
- 并发控制 / 超时策略
- 可复现的集成脚本或测试

### 适用场景

1. **把单个复杂问题切成独立子问题**
   ```
   subagent(goal="只分析 auth.py 的安全边界")
   ```

2. **上下文隔离的代码审查或调查**
   ```
   subagent(goal="审查 payment.py 的安全风险，并返回结论")
   ```

3. **主线决策前的定向探索**
   ```
   subagent(goal="找出 subagent 工具链剩余待 salvage 的文件")
   ```
