## Technical Notes

- Error handling strategy: Error events yield DoneEvent with no error field.
  This is intentional because agentkit's DoneEvent model doesn't support error
  fields. Errors should be logged separately by the caller if needed.
  
- Null safety: Text deltas that are None convert to empty string to match
  TextEvent's default value of text="".
  
- Tool call null check: Added defensive check `if event.tool_call is not None`
  before accessing tool_call attributes.
